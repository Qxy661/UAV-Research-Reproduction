# 学习笔记3：LLM 应用认知图谱 — LLM 在无人机中的能力边界

> **预计阅读：20 分钟** | **前置知识：LLM 基础概念**
> **目的**：记录 LLM 在无人机任务中的学习心得，理解能力边界

---

## 1. 认知起点：LLM 能做什么？

### 1.1 最初的兴奋

第一次看到 SayCan 和 Code as Policies 时，非常兴奋：
- LLM 可以理解自然语言任务
- LLM 可以分解复杂任务
- LLM 可以生成代码控制机器人

### 1.2 逐渐清醒

深入了解后，发现 LLM 有很多限制：
- **延迟**：API 调用需要 1-3 秒，不适合实时控制
- **可靠性**：可能生成错误的代码或不安全的动作
- **成本**：API 调用需要付费

---

## 2. LLM 的核心能力

### 2.1 任务理解

LLM 最强的能力是**理解自然语言任务**：
- "飞到 A 点然后返回" → takeoff → goto(A) → goto(home) → land
- "电量低于 20% 就返航" → check_battery → return_if_low

### 2.2 任务分解

LLM 可以把复杂任务分解为简单步骤：
- "执行网格搜索" → 计算航点 → 按顺序飞行 → 记录数据

### 2.3 异常处理

LLM 可以理解异常情况并给出建议：
- "GPS 丢失" → 切换定位方式 → 降低高度 → 悬停等待

---

## 3. LLM 的核心局限

### 3.1 延迟问题

| 模型 | 平均延迟 | P90 延迟 | 适用场景 |
|------|---------|---------|---------|
| GPT-4o | ~1.5s | ~3s | 离线规划 |
| Claude 3.5 | ~1.2s | ~2.5s | 离线规划 |
| Llama 3 70B (本地) | ~0.5s | ~1s | 近实时 |
| Llama 3 8B (本地) | ~0.1s | ~0.2s | 实时（勉强） |

**关键问题**：无人机控制频率通常 >100Hz，LLM 延迟 1-3 秒，完全无法用于实时控制。

### 3.2 可靠性问题

LLM 可能犯的错误：
- **语法错误**：生成的代码无法运行
- **逻辑错误**：代码能运行但结果错误
- **安全违规**：生成不安全的动作（如超出地理围栏）

### 3.3 成本问题

| 模型 | 价格（输入） | 价格（输出） | 100次调用成本 |
|------|------------|------------|-------------|
| GPT-4o | $2.5/1M | $10/1M | ~$0.5 |
| Claude 3.5 | $3/1M | $15/1M | ~$0.6 |
| Llama 3 70B (本地) | $0 | $0 | $0（电费除外） |

## 4. LLM 在无人机中的正确定位

### 4.1 不是控制器，是规划器

LLM 不应该直接控制无人机，而应该：
- **任务规划**：把任务分解为航点序列
- **异常处理**：分析异常情况，给出处理建议
- **人机交互**：理解操作员的自然语言指令

### 4.2 正确的架构

```
操作员 (自然语言) → LLM (任务规划) → 传统控制器 (PID/MPC) → 无人机
                        ↑
                   异常反馈 → LLM (重规划) → 传统控制器
```

### 4.3 关键领悟

> LLM 的正确位置是**决策层**，不是**控制层**。
>
> - 决策层：任务规划、异常处理、人机交互（频率 <1Hz）
> - 控制层：姿态控制、轨迹跟踪（频率 >100Hz）
>
> LLM 延迟 1-3 秒，在决策层可以接受，在控制层完全不行。

---

## 5. SayCan vs Code as Policies

### 5.1 两种范式

| 范式 | SayCan | Code as Policies |
|------|--------|-----------------|
| 输出 | 动作序列 | 可执行代码 |
| 执行 | 预定义技能库 | API 调用 |
| 灵活性 | 低（技能库有限） | 高（代码可组合） |
| 安全性 | 高（预定义技能） | 低（代码可能错误） |

### 5.2 关键领悟

> SayCan 和 Code as Policies 的区别是**灵活性和安全性的权衡**。
>
> - SayCan：安全但不灵活（只能调用预定义技能）
> - Code as Policies：灵活但不安全（代码可能错误）
>
> 在无人机场景中，安全性更重要，所以 SayCan 可能更合适。

## 6. 未来的方向

### 6.1 边缘部署

随着 LLM 模型小型化，未来可能在无人机上本地部署：
- Llama 3 8B 已经可以在 Jetson 上运行
- 延迟可以降到 0.1-0.5 秒
- 成本为零

### 6.2 多模态融合

未来的 LLM 可能直接处理视觉输入：
- GPT-4V 可以理解图像
- 可以直接从摄像头图像生成控制指令
- 省去传统的感知-规划-控制流程

### 6.3 安全保障

LLM 安全性是研究热点：
- 形式化验证：证明 LLM 输出满足安全约束
- 安全过滤器：在 LLM 输出和执行之间加一层安全检查
- 人类监督：关键决策需要人类确认

---

## 7. 学习方法论反思

### 7.1 实验驱动学习

学习 LLM 应用，最好的方式是**做实验**：
1. 设计测试任务
2. 调用 API 测试
3. 分析结果，发现边界
4. 改进 prompt 或架构

### 7.2 关键收获

> 学习 LLM 应用，关键不是知道 LLM 能做什么，而是知道**LLM 不能做什么**。
>
> - LLM 不能实时控制（延迟太高）
> - LLM 不能保证安全（可能犯错）
> - LLM 不能替代传统控制（精度不够）
>
> 理解了边界，才能正确地使用 LLM。
