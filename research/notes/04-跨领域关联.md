> **预计阅读：15 分钟** | **前置知识：三方向基础知识**

## 1. 三方向的本质区别

### 1.1 设计哲学

| 方向 | 核心问题 | 解决方案 | 信息需求 |
|------|---------|---------|---------|
| 控制 | 如何稳定跟踪？ | 反馈控制律 | 系统模型（部分） |
| RL | 如何从交互中学习？ | 策略优化 | 环境交互数据 |
| LLM | 如何理解任务？ | 语言推理 | 自然语言描述 |

### 1.2 时间尺度

```
LLM (决策层):   ~1s/决策    ← 任务规划、异常处理
RL (规划层):    ~10ms/决策  ← 轨迹规划、技能执行
控制 (执行层):  ~1ms/控制   ← 姿态稳定、轨迹跟踪
```

### 1.3 关键领悟

> 三方向不是竞争关系，而是**互补关系**。
>
> - 控制提供稳定性（高频、实时、可靠）
> - RL 提供灵活性（中频、自适应、学习型）
> - LLM 提供智能性（低频、理解、推理）
>
> 未来的飞控系统会是三者的融合。

## 2. 融合方案分析

### 2.1 LLM + 控制（SayCan 范式）

```
用户指令 → LLM (任务分解) → 航点序列 → 控制器 (PID/MPC) → 执行
```

**优势**：
- LLM 理解任务，控制器保证稳定
- 安全性高（控制器有理论保证）
- 延迟可接受（LLM 只在任务开始时调用）

**局限**：
- 灵活性有限（只能执行预定义动作）
- 无法处理未知情况

### 2.2 LLM + RL（Code as Policies 范式）

```
用户指令 → LLM (生成代码) → RL 技能库 → 执行
```

**优势**：
- LLM 理解任务，RL 提供技能
- 灵活性高（代码可组合）
- 可以处理复杂任务

**局限**：
- RL 技能需要预训练
- LLM 生成的代码可能错误

### 2.3 RL + 控制（残差学习范式）

```
参考信号 → PID (基线控制) + RL (残差补偿) → 执行
```

**优势**：
- PID 提供基线稳定性
- RL 学习补偿未建模动态
- 可以从少量数据中学习

**局限**：
- 需要设计残差的范围
- 安全性难以保证

### 2.4 全栈融合（Inner Monologue 范式）

```
用户指令 → LLM (推理) → RL (技能) → 控制器 (执行)
              ↑                              ↓
              └──── 反馈 (成功/失败) ←────────┘
```

**优势**：
- LLM 负责高层决策
- RL 负责中层技能
- 控制器负责底层执行
- 可以动态重规划

**局限**：
- 系统复杂度高
- 延迟累积
- 调试困难

## 4. 研究方法论反思

### 4.1 文献阅读

- **四遍阅读法**：Survey → Select → Critical → Synthesize
- **核心论文优先**：先读高引用、高影响力的论文
- **对比阅读**：同时读多个方向的论文，找关联

### 4.2 理论推导

- **从特殊到一般**：先推导简单情况，再推广
- **验证推导**：用代码验证公式是否正确
- **可视化**：用图表展示推导结果

### 4.3 代码复现

- **先跑通再优化**：先实现最简单的版本
- **控制变量法**：每次只改一个变量
- **多次重复**：减少随机性影响

### 4.4 实验对比

- **公平对比**：相同任务、相同计算预算
- **统计显著性**：多次重复，报告均值和标准差
- **可视化**：用图表展示对比结果

## 6. 关键收获

> 跨领域学习的核心是**找到连接点**。
>
> - 控制和 RL 的连接点：都是优化问题（一个用模型，一个用数据）
> - RL 和 LLM 的连接点：都是学习问题（一个从交互学，一个从文本学）
> - 控制和 LLM 的连接点：都是决策问题（一个在底层，一个在顶层）
>
> 找到了连接点，就能把三个方向的知识串联起来。
