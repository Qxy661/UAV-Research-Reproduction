# 学习笔记2：RL 算法认知图谱 — 从 On-policy 到 Off-policy 的理解

> **预计阅读：20 分钟** | **前置知识：强化学习基础概念**
> **目的**：记录 PPO 和 SAC 的学习心得，形成自己的理解体系

---

## 1. 认知起点：策略梯度的直觉

### 1.1 最初的理解

策略梯度的核心思想很简单：
- **好动作的概率增大**：如果某个动作带来了高奖励，就增大选择它的概率
- **差动作的概率减小**：如果某个动作带来了低奖励，就减小选择它的概率

数学表达：
```
∇J(θ) = E[∇log π(a|s) · A(s,a)]
```

### 1.2 问题：步长敏感

直接策略梯度的问题：
- **步长太大**：策略更新太激进，可能学崩
- **步长太小**：学习太慢，样本效率低
- **没有好的自适应方法**

---

## 2. PPO：用裁剪限制更新幅度

### 2.1 核心洞察

PPO 的核心洞察是：**不需要精确计算 KL 散度，只需要限制概率比 r(θ) 的变化范围**。

```
L^CLIP = E[min(r(θ)A, clip(r(θ), 1-ε, 1+ε)A)]
```

### 2.2 裁剪机制的直觉

```
当 A > 0（好动作）：
  r < 1+ε → 梯度为正，增大 π(a|s) ✓
  r ≥ 1+ε → 梯度为零，停止更新 ✋

当 A < 0（差动作）：
  r > 1-ε → 梯度为负，减小 π(a|s) ✓
  r ≤ 1-ε → 梯度为零，停止更新 ✋
```

### 2.3 关键领悟

> PPO 的本质是：**信任域方法的简化版**。
>
> TRPO 用 KL 散度约束更新步长（精确但计算复杂），PPO 用裁剪近似（简单但有效）。
>
> 这让我想到工程中的一个原则："80% 的效果来自 20% 的复杂度。"

### 2.4 On-policy 的代价

PPO 是 On-policy 算法：
- **数据只用一次**：每次更新后，旧数据就作废了
- **采样效率低**：需要大量交互数据
- **优势**：训练稳定，超参鲁棒

---

## 3. SAC：最大熵的探索

### 3.1 核心洞察

SAC 的核心洞察是：**不仅要最大化累积奖励，还要最大化策略的熵（不确定性）**。

```
J(π) = E[Σ(r + α·H(π))]
```

### 3.2 最大熵的直觉

为什么要最大化熵？
- **探索**：鼓励尝试不同的动作，避免过早收敛
- **鲁棒性**：在多个"好"策略中选择最不确定的那个
- **多模态**：可以学到多种解法，而不是只学一种

### 3.3 自动温度调节

SAC 的另一个创新是**自动调节 α**：
- α 太大：过度探索，学不到好策略
- α 太小：探索不足，陷入局部最优
- 自动调节：让 α 自适应到合适的值

### 3.4 Off-policy 的优势

SAC 是 Off-policy 算法：
- **数据可复用**：历史数据可以反复使用
- **样本效率高**：比 PPO 需要更少的交互
- **代价**：训练可能不稳定

---

## 4. PPO vs SAC 的本质区别

### 4.1 哲学差异

| 维度 | PPO | SAC |
|------|-----|-----|
| 目标 | 最大化累积奖励 | 最大化累积奖励 + 熵 |
| 探索 | 通过概率采样 | 通过熵正则化 |
| 更新 | 限制每次更新幅度 | 最大化软 Q 值 |
| 数据 | On-policy（用一次） | Off-policy（复用） |

### 4.2 适用场景

| 场景 | 推荐 | 原因 |
|------|------|------|
| 通用任务 | PPO | 稳定、简单 |
| 样本效率敏感 | SAC | Off-policy 更高效 |
| 确定性策略 | PPO | 确定性策略熵为 0 |
| 多模态任务 | SAC | 最大熵鼓励多样性 |

### 4.3 关键领悟

> PPO 和 SAC 的区别本质上是**稳定性和效率的权衡**。
>
> - PPO：牺牲效率换稳定性（On-policy，数据只用一次）
> - SAC：牺牲一些稳定性换效率（Off-policy，数据复用）
>
> 在无人机控制中，样本效率很重要（仿真成本高），所以 SAC 可能更适合。

---

## 5. RL vs 传统控制

### 5.1 本质区别

| 维度 | 传统控制 | RL |
|------|---------|-----|
| 模型 | 需要精确模型 | 不需要模型 |
| 调参 | 手工调参 | 自动学习 |
| 泛化 | 低（需重新调参） | 中（可泛化到类似任务） |
| 安全 | 可证明（Lyapunov） | 黑箱（难以证明） |
| 实时 | <1ms | <5ms（推理） |

### 5.2 融合趋势

现代飞控系统正在融合两种方法：
- **RL 学习残差**：PID 作为基线，RL 学习补偿
- **RL 优化参数**：用 RL 自动调 PID 参数
- **RL 处理复杂任务**：传统控制处理简单任务，RL 处理复杂任务

---

## 6. 学习方法论反思

### 6.1 代码驱动学习

学习 RL 算法，最好的方式是**读代码 + 跑实验**：
1. 先读论文，理解算法流程
2. 再读代码，理解实现细节
3. 然后跑实验，观察训练曲线
4. 最后改代码，验证自己的想法

### 6.2 关键收获

> 学习 RL 算法，关键不是记住公式，而是理解**设计者的权衡**。
>
> - PPO 的权衡：简单 vs 最优
> - SAC 的权衡：探索 vs 利用
> - On-policy vs Off-policy：稳定性 vs 效率
>
> 理解了权衡，就能在实际任务中做出合理的选择。
