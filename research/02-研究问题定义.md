# 研究问题定义

> 预计阅读：15 分钟 | 前置知识：01-文献综述

## 1. 研究背景

基于文献综述的发现，定义本研究的具体问题、假设和评估指标。

无人机（UAV）飞控系统是实现自主飞行的核心，其性能直接决定了飞行器的稳定性、安全性和任务执行能力。

随着控制理论和人工智能技术的发展，飞控算法的范式已经从经典PID控制扩展到自抗扰控制（ADRC）、模型预测控制（MPC），再到强化学习（RL）和大语言模型（LLM）驱动的任务规划。

然而，工程实践中面临一个关键问题：面对众多算法范式，如何为特定任务场景选择最合适的算法？

这一问题的复杂性在于：

- 不同算法范式的设计哲学截然不同，难以用单一维度衡量优劣
- 算法的理论优势未必能在工程约束下充分体现
- 缺乏统一的、系统性的对比评估框架

本研究旨在通过构建标准化的仿真评估平台，对主流飞控算法进行多维度对比分析，为工程实践提供算法选型的科学依据。

---

## 2. 总研究问题

**RQ**: 无人机飞控系统中，不同算法范式在典型任务场景下的性能-复杂度权衡如何？

### 2.1 研究动机

- **工程实践需求**：工程师在算法选型时缺乏系统性依据，往往依赖经验或文献中的零散数据
- **范式多样性**：不同范式（PID/ADRC/MPC, PPO/SAC, LLM）各有优劣，但缺乏在同一平台、同一任务下的统一评估
- **场景适配性**：需要为不同场景（悬停、抗扰、规划）提供有针对性的算法选型建议
- **可复现性**：现有对比研究往往缺乏可复现的实验代码和标准化的评估流程

### 2.2 研究目标

1. 构建基于Gym-PyBullet-Drones的标准化仿真评估平台
2. 对PID/ADRC/MPC在姿态控制任务中进行系统性对比
3. 对PPO/SAC在悬停任务中进行样本效率和最终性能的对比
4. 分析LLM在无人机任务规划中的可行性边界
5. 输出工程可用的算法选型决策树

## 3. 子研究问题

### 3.1 RQ1: 姿态控制算法对比

**问题**: PID vs ADRC vs MPC 在四旋翼姿态控制中，各维度性能如何对比？

**假设**:

- H1: ADRC在风扰抑制方面优于PID（因ESO的扰动估计能力）
- H2: MPC在约束处理方面优于PID和ADRC（因显式约束优化）
- H3: PID在实现复杂度和计算效率方面最优

**评估指标**:

| 指标 | 定义 | 测量方法 |
|------|------|---------|
| 上升时间 t_r | 响应从10%到90%的时间 | 阶跃响应曲线 |
| 超调量 σ% | (峰值-稳态)/稳态×100% | 阶跃响应曲线 |
| 稳态误差 e_ss | |设定值-稳态值| | 稳态数据 |
| 风扰最大偏差 | 风扰下的最大角度偏差 | 抗扰测试 |
| 恢复时间 t_rec | 偏差回到±5%的时间 | 抗扰测试 |
| 计算时间 t_comp | 每步控制计算耗时 | 计时器 |
| 参数敏感度 | ±20%参数偏移后性能退化率 | 参数扫描 |

**实验方案**:

1. 阶跃响应测试：施加0→1 rad/s的姿态角速率阶跃
2. 风扰抑制测试：在稳态悬停时施加阶跃风扰（5 m/s, 10 m/s）
3. 约束测试：施加执行器饱和约束（推力上限）
4. 参数敏感度扫描：对每个算法的关键参数进行±20%偏移

**调参策略**:

- PID: Ziegler-Nichols初始调参 + 手动微调
- ADRC: 带宽法调参（ω_c, ω_o）
- MPC: 权重矩阵Q/R网格搜索

### 3.2 RQ2: RL算法对比

**问题**: PPO vs SAC 在悬停任务中，样本效率和最终性能如何对比？

**假设**:

- H4: SAC的样本效率高于PPO（off-policy重用数据）
- H5: PPO的训练稳定性高于SAC（clip机制限制更新幅度）
- H6: SAC的最终性能不低于PPO（最大熵探索更充分）

**评估指标**:

| 指标 | 定义 | 测量方法 |
|------|------|---------|
| 收敛episode | 奖励首次达到阈值的episode | 训练曲线 |
| 稳态奖励 | 最后100episode平均奖励 | 训练曲线 |
| 样本效率 | 达到阈值的总交互步数 | 训练日志 |
| 训练方差 | 最后100episode奖励标准差 | 统计计算 |
| 风扰鲁棒性 | 风扰下的稳态位置误差 | 测试环境 |

**实验方案**:

1. 基础悬停训练：无风扰环境，固定目标点
2. 多随机种子训练：每算法5个随机种子，评估训练方差
3. 风扰泛化测试：训练完成后施加不同强度风扰
4. 奖励消融实验：对比密集奖励 vs 稀疏奖励

**奖励函数设计**:

```
r = -||pos - pos_target||^2 - 0.1 * ||vel||^2 - 0.01 * ||action||^2 + R_hover
```

其中 R_hover = 1.0 当位置误差 < 0.1m 时，否则为 0。

### 3.3 RQ3: LLM任务规划

**问题**: LLM在无人机任务规划中的可行性边界在哪里？

**假设**:

- H7: LLM在简单任务分解中准确率>80%（基于SayCan实验）
- H8: LLM的推理延迟(>200ms)限制了其在高频控制中的应用
- H9: 多模态反馈可以提升LLM规划的准确性

**评估指标**:

| 指标 | 定义 | 测量方法 |
|------|------|---------|
| 分解准确率 | 正确分解的任务数/总任务数 | 人工评估 |
| 异常检测率 | 正确检测的异常/总异常 | 测试集 |
| 推理延迟 | API调用耗时 | 计时器 |
| 安全违反率 | 生成不安全指令的比例 | 安全检查 |

**实验方案**:

1. 任务分解测试：设计50个不同复杂度的任务描述
2. 异常检测测试：模拟传感器故障、电量不足等场景
3. 延迟测量：统计API调用延迟的分布（均值、P95、P99）
4. 安全性评估：检查LLM是否生成危险指令（如超出地理围栏）

**任务复杂度分级**:

| 级别 | 描述 | 示例 |
|------|------|------|
| 简单 | 单步骤任务 | "飞到坐标(10,20,5)" |
| 中等 | 多步骤序列任务 | "先起飞到10m，然后飞到A点，再飞到B点" |
| 复杂 | 条件分支任务 | "如果电量>50%飞到A点，否则返航" |
| 高级 | 异常处理任务 | "巡逻A-B-C路线，遇到障碍物时绕行" |

## 4. 实验设计框架

### 4.1 控制变量法

- **独立变量**：控制算法（PID/ADRC/MPC）、RL算法（PPO/SAC）、LLM类型（GPT-4/Claude/本地模型）
- **因变量**：各项性能指标
- **控制变量**：四旋翼参数、初始条件、扰动类型、仿真步长、计算平台

### 4.2 统计方法

- 每组实验重复10次（RL实验为5个随机种子）
- 报告均值 ± 标准差
- 使用t检验或ANOVA进行显著性检验（p < 0.05）
- 效应量使用Cohen's d评估
- 置信区间使用95% CI

### 4.3 公平比较原则

- 同一动力学模型（Gym-PyBullet-Drones的quadrotor_dynamics）
- 同一评估指标体系
- 同一计算平台（CPU: Intel i5, RAM: 16GB, 无GPU）
- 同一调参预算（每算法1小时调参时间）
- 同一仿真步长（dt = 0.01s for 控制, dt = 0.02s for RL）

### 4.4 实验环境配置

| 组件 | 配置 |
|------|------|
| 仿真器 | Gym-PyBullet-Drones v1.0 |
| 动力学模型 | quadrotor_dynamics（6DoF） |
| 仿真步长 | 控制: 0.01s, RL: 0.02s |
| 风扰模型 | Dryden风模型 + 阶跃风 |
| 渲染 | PyBullet内置渲染器 |
| Python版本 | 3.10 |
| 操作系统 | Windows 11 / WSL2 Ubuntu 22.04 |

## 6. 研究范围与限制

### 6.1 研究范围

- 仅考虑多旋翼（四旋翼）平台，不涉及固定翼或旋翼机
- 控制方向仅考虑姿态环（roll/pitch/yaw），不含位置环（由外环PID处理）
- RL方向仅考虑悬停任务（hover），不含机动飞行（如flip、aggressive maneuvers）
- LLM方向仅考虑任务规划（mission planning），不含低层控制（low-level control）
- 仿真环境仅使用PyBullet物理引擎，不涉及高保真渲染（如Unreal Engine）

### 6.2 限制

- **硬件限制**：无GPU，RL训练规模受限（单episode训练时间较长）
- **验证限制**：无实飞验证，仅仿真结果，可能与实际硬件存在gap
- **模型限制**：简化动力学模型可能无法完全反映真实飞行特性
- **LLM评估限制**：依赖人工评估，存在主观性和可重复性问题
- **场景限制**：仅覆盖有限的典型场景，未涵盖所有实际飞行工况

### 6.3 威胁与应对

| 威胁 | 影响 | 应对措施 |
|------|------|---------|
| 仿真与现实gap | 结果可能无法直接迁移到实飞 | 使用经过验证的动力学模型，报告模型假设 |
| 调参不公平 | 不同算法的调参难度不同 | 固定调参预算，使用文献推荐的调参方法 |
| 随机性影响 | RL训练方差大 | 多随机种子，报告置信区间 |
| 评估主观性 | LLM评估依赖人工 | 制定明确的评估标准，多人评估取平均 |

## 附录B: 伪代码

### PID控制器伪代码

```
function PID_control(error, dt):
    integral += error * dt
    derivative = (error - prev_error) / dt
    output = Kp * error + Ki * integral + Kd * derivative
    prev_error = error
    return output
```

### ADRC控制器伪代码

```
function ADRC_control(ref, y, dt):
    # ESO: 估计状态和总扰动
    z1, z2, z3 = ESO_update(y, u, dt)
    # NLSEF: 非线性误差反馈
    e0 = ref - z1
    e1 = de0 - z2
    u0 = NLSEF(e0, e1)
    # 扰动补偿
    u = (u0 - z3) / b0
    return u
```
