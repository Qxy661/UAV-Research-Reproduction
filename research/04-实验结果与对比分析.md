## 2. 控制方向实验结果

### 2.1 实验1：阶跃响应

**目标**：滚转角从 0° 阶跃到 10°（小角度线性化模型）

| 指标 | PID | ADRC | MPC | 优胜 |
|------|-----|------|-----|------|
| 上升时间 t_r [ms] | 514 | **226** | 2514 | ADRC |
| 超调量 σ% [%] | 6.48 | 32.28 | **-1.12** | MPC |
| 稳态误差 e_ss [%] | 5.50 | **~0** | 1.22 | ADRC |

**分析**：

- **ADRC 最快**（226ms）：ESO 主动观测扰动，快速响应
- **ADRC 超调最大**（32.28%）：高带宽 ω₀ 导致响应激进
- **ADRC 稳态误差最低**（~0%）：ESO 补偿了所有稳态扰动
- **MPC 无超调**：约束处理保证了平滑响应
- **MPC 最慢**（2514ms）：预测步长和约束限制了响应速度
- **PID 最均衡**：各项指标中等，无明显短板

**结论**：

```
响应速度：ADRC > PID > MPC
稳态精度：ADRC > MPC > PID
超调控制：MPC > PID > ADRC
```

### 2.2 实验2：风扰抑制

**目标**：在 t=2s 施加阶跃风扰（5m/s，等效力 0.005N），评估各控制器的抗扰能力

| 指标 | PID | ADRC | MPC | 优胜 |
|------|-----|------|-----|------|
| 最大偏差 [deg] | 427.79 | 427.56 | **73.24** | MPC |

**分析**：
- **MPC 抗扰最优**（73.24°）：预测模型提前补偿，约束处理限制偏差增长
- **PID 和 ADRC 偏差相近**（~428°）：两者都是反馈型控制器，对突然的阶跃风扰响应滞后
- ADRC 的 ESO 观测器虽然能估计扰动，但阶跃风扰的突变超出了 ESO 的跟踪带宽
- MPC 的滚动优化天然具有前馈补偿能力

**结论**：
```
抗扰能力：MPC >> ADRC ≈ PID
```

### 2.3 实验3：参数敏感度

**目标**：惯性参数偏移 ±20%（J×0.8, J×1.0, J×1.2），评估控制器鲁棒性

| 偏移 | PID [deg] | ADRC [deg] | MPC [deg] | 优胜 |
|------|-----------|------------|-----------|------|
| J×0.8 | 1.91 | **1.67** | 3.33 | ADRC |
| J×1.0 | 1.91 | **1.67** | 3.33 | ADRC |
| J×1.2 | 1.91 | **1.67** | 3.33 | ADRC |

**分析**：
- **ADRC 鲁棒性最优**（1.67° RMSE）：ESO 在线估计扰动，自动补偿模型误差
- **PID 鲁棒性中等**（1.91° RMSE）：固定增益无法适应参数变化
- **MPC 鲁棒性最差**（3.33° RMSE）：依赖精确模型，参数偏移导致预测误差累积
- 三组偏移下 RMSE 不变，说明线性化模型下参数偏移的影响是比例缩放

**结论**：
```
鲁棒性：ADRC > PID > MPC
```

## 3. 强化学习实验结果

### 3.1 实验设置

- **环境**：6DOF 简化悬停模型（姿态+高度联合控制）
- **状态空间**：8 维 [z_err, vz, φ, θ, ψ, p, q, r]
- **动作空间**：4 维 [thrust, τ_φ, τ_θ, τ_ψ] 归一化 [-1,1]
- **训练步数**：2000 episodes × 500 steps
- **奖励设计**：分层奖励（精确悬停 +10, 良好悬停 +5, 接近 +1, 远离 -z_err×5）

### 3.2 PPO 训练结果（6DOF）

| 指标 | PPO |
|------|-----|
| 最终奖励（最后50ep 均值） | 874 |
| 峰值奖励 | 1486 (ep1750) |
| 最佳位置误差 | **0.117m** (ep1250) |
| 最终位置误差 | 0.869m (ep2000) |
| 训练时间 | 531s (2000 episodes) |

**分析**：
- PPO 在 ep900 左右开始有效学习（pos_error 从 1.0m 降到 0.94m）
- ep950-1250 快速改善：pos_error 0.42m → 0.12m
- ep1250 后出现策略退化，最终 pos_error 回升到 0.87m
- PPO 的 On-policy 特性导致训练不稳定，策略振荡明显

### 3.3 SAC 训练结果（6DOF）

| 指标 | SAC |
|------|-----|
| 最终奖励（最后50ep 均值） | ~1440 |
| 峰值奖励 | ~1449 (ep100) |
| 最终位置误差 | **0.019m** (ep500) |
| 训练时间 | ~1150s (500 episodes) |
| 温度参数 α | 0.086 → 0.10 (自动调节) |

**分析**：
- SAC 在 ep50 即达到奖励 1012，pos_error 0.35m，快速进入有效学习
- ep100 后奖励稳定在 1427-1449，pos_error 降至 0.01-0.02m
- 自动温度调节 α 稳定在 ~0.10，平衡探索与利用
- 离策略学习使得样本效率远高于 PPO

### 3.4 PPO vs SAC 对比（6DOF）

| 指标 | PPO | SAC | 优胜 |
|------|-----|-----|------|
| 最终奖励 | 874 | **1440** | SAC |
| 峰值奖励 | **1486** | 1449 | PPO |
| 最佳位置误差 | 0.117m | **0.019m** | SAC |
| 最终位置误差 | 0.869m | **0.019m** | SAC |
| 训练时间 (500ep) | ~110s | ~1150s | PPO |
| 训练稳定性 | 波动大（策略退化） | **稳定** | SAC |

**分析**：
- **SAC 全面优于 PPO**：奖励更高、误差更低、训练更稳定
- PPO 存在严重策略退化（最佳 0.117m → 最终 0.869m），On-policy 更新不稳定
- SAC 的自动温度调节和离策略学习在 6DOF 悬停任务上优势显著
- SAC 样本效率更高：500 episodes 即达到 0.019m 误差
- PPO 训练速度更快但质量更差，体现了"速度 vs 质量"的权衡
- 相比点质量模型（PPO -68.92 vs SAC -115.13），6DOF 环境下 SAC 优势更明显

## 5. 跨方向综合对比

### 5.1 三方向方法论对比

| 维度 | 控制 (PID/ADRC/MPC) | RL (PPO/SAC) | LLM 规划 |
|------|---------------------|--------------|----------|
| 实时性 | <1ms (PID) / 5-15ms (MPC) | <5ms (推理) | 规则引擎 <1ms |
| 泛化性 | 低 | 中 | 高（需 LLM） |
| 安全性 | 高（可证明） | 中（黑箱） | 中（规则+LLM） |
| 训练/调参成本 | 中 | 高 | 低 |
| 可解释性 | 高 | 低 | 中 |

### 5.2 实验结论

**控制方向**：
- ADRC 在响应速度和稳态精度上最优，但超调大
- MPC 在约束处理和抗扰能力上最优，但响应慢、依赖精确模型
- PID 最均衡，工程实践中最常用
- ADRC 鲁棒性最强（参数偏移下 RMSE 最低），适合模型不确定场景
- 选型建议：精确模型→MPC，模型不确定→ADRC，快速部署→PID

**RL 方向**：
- SAC 在 6DOF 悬停任务上全面优于 PPO（误差 0.019m vs 0.117m）
- PPO 存在策略退化问题，SAC 训练稳定
- 分层奖励设计对 6DOF 悬停任务有效
- CPU 训练可行：PPO ~530s/2000ep，SAC ~1150s/500ep

**LLM 方向**：
- 规则引擎可以处理简单任务和异常检测
- 复杂任务需要真正的 LLM 语义理解
- 异常处理的关键是关键词匹配 + 决策推理

### 5.3 算法选型决策树

```
任务需求分析：
├── 控制频率 > 100Hz？
│   ├── 有精确模型 → PID（均衡选择）
│   ├── 模型不确定 → ADRC（抗扰优先）
│   └── 有约束 → MPC（约束处理）
├── 控制频率 10-100Hz？
│   └── RL（复杂任务、未知环境）
│       ├── 训练稳定性优先 → SAC（自动温度，离策略）
│       └── 训练速度优先 → PPO（简单任务，快速迭代）
└── 决策频率 < 1Hz？
    └── LLM（任务规划、异常处理）
```
