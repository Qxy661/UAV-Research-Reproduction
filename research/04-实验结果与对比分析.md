## 5. 跨方向综合对比

### 5.1 三方向方法论对比

| 维度 | 控制 (PID/ADRC/MPC) | RL (PPO/SAC) | LLM 规划 |
|------|---------------------|--------------|----------|
| 实时性 | <1ms (PID/ADRC) | <5ms (推理) | 1-3s (API) |
| 泛化性 | 低 | 中 | 高 |
| 安全性 | 高 | 中 | 低 |
| 训练/调参成本 | 中 | 高 | 低 |
| 可解释性 | 高 | 低 | 中 |

### 5.2 算法选型决策树

```
任务需求分析：
├── 控制频率 > 100Hz？
│   ├── 是 → PID 或 ADRC（实时性优先）
│   │   ├── 有精确模型 → PID + 前馈
│   │   └── 模型不确定 → ADRC
│   └── 否 → MPC 或 RL
│       ├── 有约束 → MPC
│       └── 无约束 / 复杂任务 → RL
│           ├── 样本效率优先 → SAC
│           └── 训练稳定性优先 → PPO
└── 决策频率 < 1Hz？
    └── LLM（任务规划、异常处理）
```

### 5.3 融合方案建议

| 方案 | 适用场景 | 架构 |
|------|---------|------|
| LLM + PID | 简单任务 | LLM 规划 → PID 执行 |
| LLM + MPC | 约束任务 | LLM 规划 → MPC 执行 |
| LLM + RL | 复杂任务 | LLM 规划 → RL 执行 |
| LLM + RL + MPC | 全栈方案 | LLM 决策 → RL 技能 → MPC 约束 |
