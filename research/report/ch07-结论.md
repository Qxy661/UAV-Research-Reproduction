# 第七章 结论

## 7.1 主要发现

本研究以 Crazyflie 2.1 微型四旋翼为平台，系统对比了传统控制（PID/ADRC/MPC）、深度强化学习（PPO/SAC）和大语言模型（LLM）三类方法。主要发现如下：

### 控制方向

1. **响应速度**：ADRC（226ms）> PID（514ms）> MPC（2514ms）
2. **稳态精度**：ADRC（~0%）> MPC（1.22%）> PID（5.50%）
3. **抗扰能力**：MPC（73°）>> ADRC（428°）≈ PID（428°）
4. **鲁棒性**：ADRC（1.67°）> PID（1.91°）> MPC（3.33°）

**结论**：没有万能控制器，需根据场景选型。ADRC 综合性能最优，MPC 抗扰最强，PID 最易部署。

### RL 方向

1. SAC 在 6DOF 悬停任务上全面优于 PPO（误差 0.019m vs 0.117m）
2. PPO 存在策略退化问题（0.117m → 0.869m），SAC 训练稳定
3. 分层奖励设计对 6DOF 悬停任务有效

**结论**：SAC 是无人机悬停任务的更优选择，但训练成本更高。

### 跨方向对比

1. PID 在无风环境中表现优异（0.0127m），优于 SAC（0.019m）
2. PID 在风扰下性能急剧退化（0.78m @ 5m/s，61 倍）
3. SAC 的风扰鲁棒性有待进一步验证

**结论**：在已知稳定环境中，PID 是更优选择；在未知/动态环境中，RL 可能更有优势。

### LLM 方向

1. 规则引擎可处理简单任务（75%）和异常检测（100%）
2. 复杂任务需要 LLM 的语义理解能力
3. API keys 不可用限制了真实 LLM 评估

**结论**：LLM 在无人机任务规划中有潜力，但需要真实 API 验证。

## 7.2 贡献总结

1. 建立了完整的三方向研究框架（文献→理论→复现→实验）
2. 实现了 PID/ADRC/MPC/PPO/SAC 的全栈代码复现
3. 设计了跨方向统一基准，首次在同一环境中对比传统控制与 RL
4. 提出了 6DOF 分层奖励环境，SAC 达到 0.019m 位置误差
5. 系统评估了 LLM 在无人机任务规划中的能力边界

## 7.3 展望

未来工作方向：

1. **更真实的环境**：Dryden 湍流模型、12 状态完整模型、传感器噪声
2. **更复杂的任务**：轨迹跟踪、避障、编队飞行
3. **RL 改进**：PPO 正则化、SAC 风扰训练、Sim-to-Real 迁移
4. **LLM 集成**：接入 GPT-4o/Claude，实现 LLM+控制的混合架构
5. **工程落地**：代码优化、单元测试、部署到真实硬件
