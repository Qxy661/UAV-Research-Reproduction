# 第六章 讨论

## 6.1 算法选型决策

基于实验结果，提出算法选型决策树：

```
任务需求分析：
├── 控制频率 > 100Hz？
│   ├── 有精确模型 → PID（均衡选择）
│   ├── 模型不确定 → ADRC（抗扰优先）
│   └── 有约束 → MPC（约束处理）
├── 控制频率 10-100Hz？
│   └── RL（复杂任务、未知环境）
│       ├── 训练稳定性优先 → SAC
│       └── 训练速度优先 → PPO
└── 决策频率 < 1Hz？
    └── LLM（任务规划、异常处理）
```

### 6.1.1 控制方法选型

| 场景 | 推荐方法 | 原因 |
|------|----------|------|
| 快速部署 | PID | 结构简单，调参直观 |
| 模型不确定 | ADRC | ESO 在线补偿，鲁棒性强 |
| 有安全约束 | MPC | 约束处理，无超调 |
| 强风扰环境 | MPC | 预测优化，抗扰最优 |

### 6.1.2 RL 方法选型

| 场景 | 推荐方法 | 原因 |
|------|----------|------|
| 训练稳定性 | SAC | 自动温度调节，无策略退化 |
| 样本效率 | SAC | Off-policy，replay buffer |
| 快速原型 | PPO | 实现简单，训练快 |
| 复杂任务 | SAC | 最大熵探索，泛化性强 |

### 6.1.3 PID vs SAC

跨方向对比揭示了一个重要发现：

- **无风环境**：PID（0.0127m）优于 SAC（0.019m），且训练成本为零
- **有风环境**：PID 性能急剧退化（0.78m @ 5m/s），SAC 的鲁棒性待验证
- **工程意义**：在已知环境中，PID 是更优选择；在未知/动态环境中，SAC 可能更有优势

## 6.2 局限性

### 6.2.1 控制方向

- 仅测试了小角度线性化模型（10°），未验证大角度非线性行为
- 风扰模型过于简单（阶跃力），未使用 Dryden 等真实湍流模型
- MPC 使用单通道解耦 QP，未考虑多轴耦合

### 6.2.2 RL 方向

- 6DOF 简化模型与真实 12 状态模型有差异
- PPO 存在策略退化，可能需要更好的正则化
- SAC 风扰训练尚未完成
- 仅在悬停任务上测试，未验证轨迹跟踪等复杂任务

### 6.2.3 LLM 方向

- 仅使用规则引擎，未接入真正 LLM API
- 测试用例有限（20 个任务 + 10 个异常场景）
- 评估指标单一（准确率），未考虑响应质量

### 6.2.4 跨方向对比

- PID 和 SAC 的物理模型不完全一致（12 状态 vs 8 状态）
- 跨方向对比仅限于悬停任务，未扩展到轨迹跟踪
